移动应用程序"Plan"技术规范

1\. 概述

该应用程序允许用户上传照片，使用坐标网格在照片上标记交互区域（按钮），并为每个区域附加文本评论、照片和视频。应用程序运行于
Android 14 及以上版本，使用 Kotlin 在 Android Studio
中开发，源代码托管在 GitHub 上。

2\. 项目管理

项目 是内容单元，包含：

主照片，

一组区域（按钮），

附加到区域的媒体文件和文本。

项目字段：

名称（必填），

类型1（文本，可选），

类型2（文本，可选），

描述（文本，可选），

备注（文本，可选）。

创建项目：

用户从图库选择照片或使用相机拍摄。

获取照片后，打开表单，名称必填，其他字段可选。

保存后，项目出现在主屏幕列表中，应用程序切换到项目屏幕并进入编辑模式（网格）。

编辑项目（仅字段：类型1、类型2、描述、备注）：

在主屏幕上选中项目后，通过"铅笔"按钮进行编辑。

名称和照片不可编辑。

删除项目：

通过项目屏幕菜单中的"删除"项，确认后删除，并返回主屏幕。

也可通过主屏幕上的编辑卡片删除项目。

3\. 主屏幕（屏幕1）

3.1. 头部

项目名称搜索框（实时过滤列表）。

右侧菜单按钮（四条横线）。菜单包含：

Добавить / Add / 添加 --- 创建新项目（选择照片，然后填写字段）。

Импорт / Import / 导入 --- 从 ZIP 压缩包导入项目。

Обмен / Share / 分享 --- 通过 Wi‑Fi/蓝牙接收和发送项目。

Экспорт / Export / 导出 --- 导出到 PC（生成 HTML +
媒体文件夹）。若已选中项目则导出该项目，否则弹出对话框选择：导出所有项目或从列表中选择一个。

Настройки / Settings / 设置 ---
选择语言（俄语、英语、汉语）和主题（系统、亮色、暗色）。

Выход / Exit / 退出 ---
关闭应用程序，若打开的窗口有未保存更改则提示确认。

3.2. 主体

项目列表。每个项目：左侧项目名称，右侧照片缩略图。

双击名称选中项目（高亮）。选中项目时，底部栏按钮（星标、铅笔、对勾）变为可用。

单击缩略图打开照片全屏查看（无标记）。

3.3. 底部栏

始终可见，考虑系统导航栏内边距。四个按钮：

Домик / Home / 首页 --- 主屏幕（取消项目选中）。

Звезда / Star / 星标 ---
仅在选中项目时可用。点击后打开项目屏幕的查看模式。

Карандаш / Pencil / 编辑 ---
仅在选中项目时可用。打开编辑项目字段的对话框（类型1、类型2、描述、备注）。对话框中还包含"删除项目"按钮。保存或删除后返回主屏幕，选中状态清除。

Галочка / Checkmark / 查看 ---
仅在选中项目时可用。打开只读对话框查看项目所有字段。

4\. 项目屏幕（屏幕2）

4.1. 头部

当前项目名称。

搜索栏 ---
搜索区域（按钮）名称。输入文本时，名称包含搜索文本的区域会在照片上高亮（例如亮色边框）。

右侧菜单按钮（三个点或四条横线）。菜单包含：

Редактировать / Edit / 编辑 --- 切换进入编辑模式（网格）并退出。

Экспорт / Export / 导出 --- 将当前项目导出到 PC（HTML + 文件夹）。

Поделиться / Share / 分享 --- 通过
Wi‑Fi/蓝牙将当前项目发送到另一设备（ZIP 压缩包）。

Удалить / Delete / 删除 --- 确认后删除当前项目，之后返回主屏幕。

Очистить / Clear / 清除 ---
确认后删除当前项目的所有区域（区域及其内容删除，项目保留）。

4.2. 主体

照片支持缩放（双指捏合）和平移。

照片上方根据模式显示区域（按钮）或网格。

4.3. 底部栏

Домик / Home / 首页 ---
返回主屏幕。若存在未保存更改（编辑模式或打开的区域卡片有更改），则显示确认对话框。

Звезда / Star / 星标 --- 在屏幕2上不使用（禁用）。

Карандаш / Pencil / 编辑 ---
当区域卡片打开时可用。点击将卡片切换到编辑模式。

Галочка / Checkmark / 保存 ---
当区域卡片打开且已进行更改时可用。点击保存更改（显示确认对话框）。

4.4. 查看模式

区域以所选颜色的半透明填充显示（透明度 50%）。

单击区域无操作。

双击区域打开区域卡片。

4.5. 区域卡片

以可滚动的 BottomSheetDialogFragment 打开。包含：

媒体画廊（轮播）显示附加的照片和视频。点击元素进入全屏查看：照片可缩放、滑动切换媒体；视频可播放。

字段：名称（必填）、类型1、类型2、状态（选择现有或新建）、描述、备注。

按钮 Добавить фото / Add photo / 添加照片 和 Добавить видео / Add video
/ 添加视频 --- 打开图库/相机选择。

卡片查看模式：

所有字段只读。

底部栏按钮"铅笔"和"对勾"可用。

卡片编辑模式：

通过点击"铅笔"打开。

字段变为可编辑，出现"保存"按钮（或底部栏"对勾"）。

保存时数据更新到数据库；照片上区域颜色根据所选状态变化。

关闭卡片时若有未保存更改，显示对话框选项：保存、不保存、取消。

状态：

从已有状态（名称+颜色）中选择，或新建状态。

状态颜色为 7 种经典色：红、橙、黄、绿、青、蓝、紫。

新建状态时，用户输入名称并选择 7 种颜色之一。

4.6. 编辑模式（网格）

通过菜单项"编辑"进入/退出。

照片上方显示正方形坐标网格。现有区域隐藏。

网格单元格大小通过滑块和"+""-"按钮调节，按钮位于底部栏上方。仅当项目尚无区域时才能更改单元格大小。若已有区域，更改时会警告将删除所有区域，用户需确认或取消。

可用单元格大小为相对于原始照片尺寸的 2 的倍数（1、2、4、8 等）。

创建区域：

双击单元格开始新选择（清除之前选择）。

单击单元格添加到选择，仅当该单元格与已选单元格相邻（边或对角线）。

最后添加的单元格可通过再次单击移除。

完成选择后，点击"创建"（或"对勾"）。打开对话框：

区域名称（必填）。

选择状态（现有或新建）。

字段：类型1、类型2、描述、备注（可选）。

保存后创建区域，选择清除。

删除所有区域 --- 通过菜单项"清除"。

5\. 数据库

存储使用 Room。实体：

projects：id (Long), name (String), photoUri (String), type1 (String?),
type2 (String?), description (String?), note (String?), cellSize (Int),
createdAt (Long), updatedAt (Long)。

states：id (Long), name (String), color (Int)。预填充 7 种颜色。

regions：id (Long), projectId (Long), name (String), stateId (Long?),
type1 (String?), type2 (String?), description (String?), note (String?),
cellsJson (String), createdAt (Long), updatedAt (Long)。cellsJson
存储单元格数组，格式为 \[{\"row\":0,\"col\":0}, \...\]。

contents：id (Long), regionId (Long), type (String ---
\"text\"/\"photo\"/\"video\"), data (String), sortOrder (Int), createdAt
(Long)。对于文本，data 存储文本；对于照片/视频，存储相对路径。

6\. 项目导入与导出

6.1. 导出到另一设备（ZIP）

生成 ZIP 压缩包，结构如下：

project.json（项目字段，不含 id 和 photoUri）。

photo.jpg（原始照片）。

文件夹 regions/，内含每个区域的 JSON
文件，包含状态信息（名称和颜色）以及内容列表（媒体相对路径）。

文件夹 media/，内含所有附加的照片和视频。

6.2. 从 ZIP 导入

应用程序解压压缩包，创建新项目，恢复状态（根据名称和颜色链接到现有状态或创建新状态），将媒体文件复制到私有存储。导入后项目出现在列表中。

6.3. 导出到 PC（HTML + 文件夹）

创建以项目名称命名的文件夹，包含：

photo_with_areas.jpg ---
原始照片上叠加区域（边框及状态颜色的半透明填充）。

每个区域的子文件夹，将附加的媒体文件复制到其中，并创建
comment.txt（文本评论，若有）。

report.html --- 表格，列出所有区域、字段及文件链接。

7\. 设备间共享

通过 Wi‑Fi Direct 或蓝牙传输项目。发送端按导出格式生成 ZIP
压缩包并发送。接收端自动识别压缩包，运行导入，将项目添加到列表。

8\. 非功能性需求

系统导航栏适配：底部栏增加内边距，避免被导航按钮遮挡。

键盘：所有文本输入屏幕使用 adjustResize，确保输入字段不被键盘遮挡。

确认提示：任何可能导致数据丢失的操作（删除项目、清除区域、未保存退出）均显示对话框，选项包括：保存、不保存、取消（适用时）。

本地化：所有界面文字均提取到资源文件，支持三种语言：俄语、英语、汉语。

免费库：使用的所有第三方库必须具有开放许可证（Apache 2.0、MIT 等）。

9\. 架构与技术栈

架构：MVVM 与 Clean
Architecture（数据层、领域层、表现层）。使用仓库和用例。

管理器（单例）：ProjectManager、GridManager、ExportManager、TransferManager
--- 用于集中管理状态和复杂操作。

语言：Kotlin。

UI：Jetpack Compose（也可使用 XML 自定义视图，由开发者选择）。

数据库：Room。

照片缩放：PhotoView。

相机：CameraX。

图片加载：Glide 或 Coil。

Wi‑Fi Direct / 蓝牙：Android 内置 API。

ZIP：内置 ZipOutputStream / ZipInputStream。

协程：Kotlin Coroutines。

10\. 开发阶段

项目设置，添加依赖，创建目录结构。

实现数据库（实体、DAO、仓库、预填充状态）。

开发主屏幕（项目列表、创建、编辑、删除、搜索）。

实现项目屏幕（基础功能，不含网格）：照片显示、加载区域、绘制填充、区域搜索、区域卡片（查看、编辑字段、添加媒体、轮播、全屏查看）。

实现编辑模式（网格）：显示网格、更改单元格大小、选择相邻单元格、创建区域、删除所有区域。

实现 ZIP 导入/导出（设备间传输）及导出到 PC（HTML + 文件夹）。

实现 Wi‑Fi Direct / 蓝牙共享。

本地化、设置（语言、主题）。

测试与调试。
